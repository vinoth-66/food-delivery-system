Food Delivery System (Console-Based Java Application)

Problem Statement
The objective of this project is to develop a food delivery system that simulates real-world food ordering operations. The system manages restaurants and their food items, allows users to browse available items, add items to a cart, place orders, and handle delivery operations.

Approach / Logic Used
This project follows a layered architecture for better structure and maintainability.

Model Layer:
Represents entities such as User, Restaurant, FoodItem, Order, and DeliveryAgent.

Repository Layer:
Handles data storage using collections.

Service Layer:
Contains business logic.

Controller Layer:
Manages user interaction through the console.

Functional Flow

Restaurant Operations:
Add food items
Remove food items
Update item price

User Operations:
View all available food items
Add items to cart
View cart
Place order
Cancel order

Order and Delivery Flow:
Order is created from the cart
Delivery agent is assigned to the order
Delivery agent can accept or cancel the order
